# 一般操作
首先，按下 `Super`+`/` 查看快捷鍵列表。

它也包含一些啟動器的 Actions。

## 總覽 / 啟動器
按下 `Super` 或 `Super`+`Tab` 開啟。

- **視窗**
  - 拖曳以將它們移動到不同的工作區
  - 右鍵點擊以查看一些選項
- **搜尋**
  - <u>計算</u>：輸入以數字開頭的內容
  - <u>指令</u>：如果你輸入，它們會被偵測並顯示給你。
    - 預設執行 headless 執行
    - 如果指令以 sudo 開頭，它將在 `foot` 終端機中啟動。
  - <u>目錄搜尋</u>：輸入以 `~` 或 `/` 開頭的目錄。
    - 如果路徑正確，子目錄會被顯示。啟動會開啟資料夾 / 檔案
    - 直接按 Enter 開啟輸入的資料夾 / 檔案
  - <u>Actions</u>：每個 action 以 `>` 開頭。請參閱**快捷鍵速查表**查看列表
  - <u>Apps</u>：當然是預設的！

## 工作列控制
- 媒體：
  - 右上角
    - 中鍵點擊：播放 / 暫停媒體
    - 右鍵點擊：下一首
    - 捲動：變更音量
  - 點擊音樂膠囊以開啟控制視窗
- 亮度：捲動左上角
- 啟動器 / 總覽：右鍵點擊工作區小工具
- 虛擬鍵盤：中鍵點擊工作區小工具
- 複製通知中的文字：按住約 800 毫秒（參考：[#224](https://github.com/end-4/dots-hyprland/issues/224#issuecomment-1923706599)）
- 工作區：
  - 捲動以切換工作區
  - 點擊工作區以聚焦
  - 點擊側鍵（遊戲滑鼠上）以切換特殊工作區（scratchpad）


# API

_位於左側邊欄_

## AI 服務
- 每個 AI 服務都需要 API 金鑰
  - 如果你沒有，當你點擊它時會出現提示，帶你到提供者的網站
  - 當你在那個網站上按照說明取得金鑰後，只需將金鑰貼上並傳送到側邊欄聊天框中，它就會被儲存。上述提示應該會消失，你應該就能使用 API 了
  - 如果你需要變更金鑰，使用 `/key` 指令。它會告訴你該怎麼做。

## Waifu 服務
- 不需要 API 金鑰，耶！
- 對於「Waifus」分頁，輸入一個類別。查看其文件以取得列表：https://waifu.pics/docs
- 對於「Anime Booru」分頁：
  - `/mode yandere` 切換到 yande.re 模式（一般圖片）【預設】
  - `/mode konachan` 切換到 Konachan（桌布）
  - 輸入空格分隔的標籤，你會得到 20 張圖片（預設值，可設定）
  - 包含一個數字以跳轉到該頁
  - <sub><sub><sub>你可能會問，為什麼不直接使用網頁介面？因為 API 預設停用了篩選器</sub></sub></sub>

# 桌布與配色
- 要選擇桌布，請按下 `Ctrl`+`Super`+`T` 並選擇一張圖片。小工具、Gtk 應用程式（重新啟動後）和 Qt 應用程式（請參閱[如何正確設定](https://ii.clsty.link/en/ii-ags/01setup/#not-so-optional-stuff)）將使用 Material 配色主題。
- 如果你想要更鮮豔或更柔和的配色，可以變更 Material 調色盤
  - 按下 `Super`+`,` 開啟調色盤面板。點擊其右下角的小箭頭以查看選項。
  - 選擇所需的調色盤，如 Tonal Spot（正常的）、Neutral（更柔和的配色）或 Vibrant...

# 工作區群組

## 簡介

預設情況下，工作列最多顯示 10 個工作區。然而，許多使用者發現自己需要超過 10 個工作區的限制。隨著開啟的應用程式數量增加，雜亂的工作區或重新排列視窗變得麻煩。

工作區群組引入了一個靈活的系統，允許無限數量的工作區（在 Hyprland 的限制內），組織成群組以便於導覽和管理。

- **主要功能**
  - **可擴展的工作區**：超越 10 個工作區的限制，而不會使工作列或總覽小工具變得雜亂。根據需要建立盡可能多的工作區，而不會影響組織性。
  - **工作區群組**：工作區自動分為每 10 個一組（例如 1-10、11-20、21-30）。活動工作區決定工作列（以及總覽小工具）中可見的群組。
  - **快捷鍵支援**：Hyprland 快捷鍵已設定為無縫適應此功能
    - 按下 `Super` + `3` 會將你移動到工作區 3（如果你當前在工作區 1-10），或工作區 13（如果你當前在工作區 11-20）

- **優勢**
  - **多工處理大師**：透過為特定情境分配工作區群組，輕鬆在不同任務或專案之間切換
  - **整潔的環境**：即使有大量工作區，也能保持乾淨有序的桌面體驗
  - **多螢幕威力**：建立螢幕專屬的工作區群組，打造受 [awesome-wm](https://awesomewm.org/) 啟發的工作流程
    - 例如 monitor-1 中的工作區 1-10、monitor-2 中的工作區 11-20、monitor-3 中的工作區 21-30，依此類推。

## 導覽與工作區管理

- **在群組內導覽 / 移動工作區**：標準 Hyprland 快捷鍵適用。
  - `Super` + `2` → 工作區 **2**（如果你在工作區 **1-10，即群組 1**）。
  - `Super` + `2` → 工作區 **12**（如果你在工作區 **11-20，即群組 2**）。
  - `Super` + `Alt` + `3` 靜默移動聚焦的視窗 → 工作區 **3**（如果你在工作區 **1-10，即群組 1**）。
  - `Super` + `Alt` + `3` 靜默移動聚焦的視窗 → 工作區 **13**（如果你在工作區 **11-20，即群組 2**）。

- **在群組之間導覽**：
  - 在當前群組的最後一個工作區之外導覽以在下一個群組中產生新的工作區。例如，按下 `Super` + `0` 移動到工作區 10，然後按下 `Control` + `Super` + `Right` 移動到下一個工作區，即下一個群組中的 11。
  - 或者，你可以在工作區列上捲動游標以快速導覽到下一個群組。
  - 建議新增以下 Hyprland 快捷鍵以跳轉到相鄰的工作區群組
    ```ini title="~/.config/hypr/hyprland/keybinds.conf"
    bind = SUPER ALT, Z, workspace, r-10
    bind = SUPER ALT, X, workspace, r+10
    ```
- 總覽小工具也提供多功能的視窗和工作區管理。使用 `SUPER` 在群組中查看所有視窗和工作區，在該群組的工作區之間拖放視窗，右鍵點擊任何視窗以存取額外選項。
:::tip
如果你想修改快捷鍵或為工作區導覽加入[更多功能](https://wiki.hyprland.org/Configuring/Dispatchers/)，請使用 `${HOME}/.config/ags/scripts/hyprland/workspace_action.sh` 腳本，而不是 `hyprctl dispatch`。這將使用活動工作區 ID 確定當前的工作區群組，並分派到正確的工作區。
:::

## 多螢幕注意事項

請考慮以下策略以有效管理多螢幕：
- 將群組 1（工作區 1-10）分配給主螢幕，群組 2（工作區 11-20）分配給次要螢幕。
  - 啟動時，使用 `Super` + `0`，然後 `Ctrl` + `Super` + `Right` 手動將次要螢幕上的起始工作區移動到第二群組（例如工作區 11）。
  - 實際上，這也會為每個螢幕建立獨立的總覽小工具。
- 使用[工作區綁定](https://wiki.hyprland.org/Configuring/Workspace-Rules/#rules)將特定工作區始終放置在特定螢幕上。使用 `hyprctl monitors | grep Monitor` 取得所有螢幕名稱。
  ```ini title="~/.config/custom/general.conf"
  # 將工作區 1-10（群組 1）綁定到主螢幕
  workspace = 1, monitor:eDP-1, default:true
  workspace = 2, monitor:eDP-1, default:true
  ...

  # 將工作區 11-20（群組 2）綁定到次要螢幕
  workspace = 11, monitor:HDMI-A-1, default:true
  workspace = 12, monitor:HDMI-A-1, default:true
  ...
  ```
- 同時在每個螢幕上聚焦兩個工作區，並根據需要在螢幕 / 群組之間移動 / 交換視窗（使用 `Super` + `Left/Right/Up/Down` 或 `Super` + `Shift` + `Left/Right/Up/Down`）。

:::note
工作區群組並非由 Hyprland、AGS 或任何狀態列程式如 Waybar / Polybar 原生支援。此設定透過調整 Hyprland 分派器和少量自訂 AGS 模組來嘗試實現。如果你覺得某些元素可以進一步改進，歡迎開啟 [Discussion](https://github.com/end-4/dots-hyprland/discussions) 或提交 PR。
:::

# 專注模式

- 工作列的寬度將減半
- 只會顯示工作區指示器，不顯示數字
