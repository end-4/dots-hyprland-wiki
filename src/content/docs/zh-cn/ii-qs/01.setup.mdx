# 安装
### 贴士
- 这里**只提供用户级别的更改**。你需要负责系统基本功能的其他软件/配置。
- **不需要重新安装你的系统**。

### 安装方式
- [自动化安装](#自动化安装): 使用脚本
- [手动安装](#手动安装)：手动克隆与复制
- [社区分支](#分支): 针对特定发行版的安装
- [第三方发行版](#第三方发行版): 开箱即提供“illogical-impulse”特性的操作系统

:::tip[从 AGS 版本迁移]
*如果你没有使用旧的 AGS 版本，或者不使用 Arch Linux，则可以跳过此步骤*

Quickshell 版本在安装后应该可以正常工作，但在安装新版本之前，你可以卸载旧包：

```sh
sudo pacman -Rs $(pacman -Q | grep "illogical-impulse-" | awk '{print $1}')
```

然后执行安装。
:::

## 自动化安装

:::caution[Hyprland 0.55?]
如果您的发行版尚未发布 Hyprland 0.55 和/或您还没有准备好，您应该切换到 Pre-Hyprland Luaification 版本。安装示例：
```
git clone --recursive https://github.com/end-4/dots-hyprland && cd dots-hyprland && git checkout 2026.05.11 && ./setup install
```
:::

### 发行版支持
该脚本在继续执行前会自动检测发行版。
以下是关于发行版支持情况的概览。
- **开发者支持：**
  - [Arch Linux](https://archlinux.org/) 和 [EndeavourOS](https://endeavouros.com/) 是最有力支持的
  - [CachyOS](https://cachyos.org/) 应该能正常工作，不过在极少数情况下，软件包的差异可能会给缺乏经验的用户带来问题。基于 Arch 的其他发行版也是如此。
- **由社区支持：**
  - [Gentoo](https://www.gentoo.org/) (查看 [sdata/dist-gentoo/README.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/dist-gentoo/README.md) 以获取更多信息).
  - [Fedora](https://fedoraproject.org/) (查看 [sdata/dist-fedora/README.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/dist-fedora/README.md) 以获取更多信息).
  - 其他发行版：欢迎贡献！请参阅 [this page](../../dev/inst-script/#help-wanted) 如果你愿意帮忙的话。
- **制作中：**
  - 目前正在开发中的 `dist-nix` 旨在提供跨发行版支持。对于像 [Debian](https://www.debian.org/) 这样软件包版本相对较旧的发行版来说，这几乎是安装 illogical-impulse 的唯一途径。欢迎查看 [#2382](https://github.com/end-4/dots-hyprland/discussions/2382) 进行尝试。

请使用以下任一方法。前者是后者的一个更便捷的封装，尽管有些人对此存疑。

## 手动安装

适用于几乎任何发行版

:::caution
此节仅用于演示。

强烈不推荐使用，因为它会浪费你大量时间。
:::

### 克隆并进入目录

```bash
t=~/.cache/dots-hyprland   # 别让我们把你的主文件夹弄得一团糟
git clone https://github.com/end-4/dots-hyprland.git "$t" --filter=blob:none --recurse-submodules
cd "$t"
```

### 安装依赖包

对于 Arch 系发行版：
- 安装 `./sdata/dist-arch/*/PKGBUILD` 文件中 `depends` 数组所列出的所有软件包。只需使用像 `yay` 这样的 AUR 辅助工具即可。
  - 你也可以安装你所发现的合适的替代品。

对于非 Arch 发行版：
- 找出 [deps-info.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/deps-info.md) 中所列软件包的对应版本并进行安装。
- 如果您已成功在非 Arch（或基于 Arch）的 Linux 发行版上完成安装，欢迎提交 PR。详情请参阅[此页面](../../dev/inst-script/#help-wanted)。

### 权限/服务设置等

阅读 `sdata/subcmd-install/2.setups.sh` 的内容。
- 如果你认为它没问题，那么只需运行 `./setup install-setups` ，这样就能运行此脚本。
  - 不要直接运行 `sdata/subcmd-install/2.setups.sh`，因为它是为了被 `./setup` 引用而设计的。
- 或者，手动运行此脚本中的行并进行调整。

### 复制配置文件
- 将 `dots/*` 复制到 `$HOME/`（在执行前请先查看，以免意外覆盖你自己的配置文件）
- 或者，阅读 `sdata/subcmd-install/3.files.sh`。
  - 如果你认为它没问题，那么只需运行 `./setup install-files` ，这样就能运行此脚本。
  - 不要直接运行 `sdata/subcmd-install/3.files.sh`，因为它是为了被 `./setup` 引用而设计的。

### 完成
- 完成后，重新登录 Hyprland。
  - `Ctrl`+`Super`+`T` 选择壁纸。
  - `Super`+`/` 查看快捷键列表。玩得开心！

# 安装后
## 不那么可选的东西

### 防止通知守护进程冲突
- 诸如 `dunst` 和 `mako` 之类的通知守护进程可能是您所用发行版预装的组件，它们可能会干扰 Quickshell 的运行。如果您在其他地方不使用这些程序，建议将其卸载。

### 避免 AUR 冲突（仅限 Arch（及其衍生发行版））
问题所在：请参阅[此评论](https://github.com/end-4/dots-hyprland/discussions/3165#discussion-9788226)。

解决方案：手动编辑 `/etc/pacman.conf` 文件（由于自动修改该文件存在一定风险，我们不进行自动操作），并添加
```conf
IgnoreGroup=illogical-impulse
```
关于 Pacman 的配置，也可参阅 [Arch Wiki - Pacman](https://wiki.archlinux.org/title/Pacman#Skip_package_group_from_being_upgraded)。

## 启动 Hyprland

- 要启动 Hyprland，你可以使用 DM（显示管理器）或直接使用 `tty`。
- Hyprland wiki 建议使用 uswm-managed 的会话启动 Hyprland，但我们不这样做。使用这个不会破坏 dotfiles，但你可能会从其他桌面环境获得自动启动的垃圾文件（例如重复的身份验证对话框）

详见 [Hyprland wiki](https://wiki.hypr.land/Getting-Started/Master-Tutorial/#launching-hyprland)。以下是一些额外提示。

### 如何在 `tty1` 登录后自动启动 Hyprland？
对于 ZSH 或 BASH，在你的 `~/.zshrc` 或 `~/.bashrc` **底部**添加此行：
```bash
source ~/.config/zshrc.d/auto-Hypr.sh
```

对于 FISH，在你的 `~/.config/fish/config.fish` **底部**添加此行：
```fish
source ~/.config/fish/auto-Hypr.fish
```

附注：如果你希望通过 tty 启动 Hyprland，建议禁用 DM。

### 我是新手。什么是 tty 和 DM？

以下是简要介绍，仅供快速理解，但**不完全准确**。

你可以将 `tty` 视为 Linux 系统的某种“底层”
通常有 7 个 `tty`：`tty1` 到 `tty7`。你可以按 `Ctrl+Alt+F<n>` 切换到 `tty<n>`，然后输入用户名和密码登录。

登录后，你能够通过命令启动图形环境，例如 `Hyprland`。
一般而言，图形界面只能在登录**之后**启动。

但是如果我们想要登录界面本身就有图形界面呢？

于是 DM（Display Manager 即显示管理器，也称为 "LM"，即 Login Manager，登录管理器）出场了。
- 一些常见的 DM：
  - `sddm`：常用于 KDE Plasma。
  - `gdm`：常用于 Gnome。
- 它在系统级别启用，并在系统启动后自动启动（在还未登录时）。
  - 在基于 systemd 的发行版上，DM 通常作为 systemd 服务启用。运行以下命令查看启用了哪个 DM。
    ```bash
    grep 'ExecStart=' /etc/systemd/system/display-manager.service
    ```
    如果返回 `No such file or directory`，则没有启用 DM，或者这不是基于 systemd 的发行版。
- 它为你提供图形界面登录并选择图形环境（例如 Hyprland）。
  - DM 如何知道哪些图形环境可用？
    - 通常它会在路径 `/usr/share/xsessions` 中搜索 X11 的图形环境，以及在 `/usr/share/wayland-sessions` 中搜索 Wayland 的图形环境。
    - 这些目录下的 desktop 文件包含图形环境的信息。

# 更新

:::caution[Update to Hyprland 0.55]
Hyprland 0.55 引入了基于 Lua 的配置方式，以取代原有的 Hyprlang。尽管旧格式仍可使用，但由于部分变量位置的变动，已出现了一些破坏性变更。Hyprland 仅提供两个月的过渡期供用户迁移至 Lua 配置。

- 如果您决定暂时继续使用 0.54 版本，请向上滚动查看“自动安装”章节中的“Hyprland 0.55?”提示。
- 如果您准备升级到 0.55 版本，请按以下步骤操作：
  - 将 `~/.config/hypr/custom` 移动到其他位置，并删除该文件夹 
  - 照常运行 `git pull` 和 `./setup install`
  - 参考旧配置文件以及 dotfiles 提供的最新配置文件，将您的旧配置改写到新的 .lua 文件中
:::

:::note[TL;DR]
如果您在安装 Arch（或基于 Arch 的）发行版时使用了在线安装脚本，可以运行以下命令进行更新：
```bash
cd ~/.cache/dots-hyprland && git stash && git pull && ./setup install
```
:::

目前我们既没有更新检查器，也没有自动更新系统。由于安装脚本被设计为幂等的（idempotent），若要更新 dotfiles，您需要在从仓库拉取最新更改后，再次运行安装脚本。
1. 进入仓库目录。默认路径应为 `~/.cache/dots-hyprland`，除非您将仓库克隆到了其他位置。
2. 运行 `git stash` 以保存您的本地更改。理想情况下，安装过程不应修改由 Git 追踪的本地文件，但可能会发生意外情况，因此在拉取最新更改前，您需要先暂存（stash）这些更改。
3. 运行 `git pull` 以获取并拉取仓库中的最新更改。
4. 再次运行 `./setup install`。 
- 跳过您不希望脚本更新的步骤（特别是涉及 `rsync ...` 的步骤，因为它们会**覆盖目标路径下的文件**）。通常，您可能希望执行涉及 `dots/.config/quickshell` 的 `rsync` 步骤。

:::tip
运行 `./setup install -h` 以查看更多用法。
:::

:::caution[如果您是手动安装的]
尽管极不推荐，但如果你是手动安装的，请按以下步骤操作：
- 进入仓库目录，运行 `git pull && git submodule update --init --recursive` 以获取最新更改。
- 务必再次确认所有依赖项均已安装，因为更新可能涉及新的依赖项。
- 获取你需要的文件。通常，你可能需要获取 `dots/.config/quickshell` 文件夹。
:::

# 卸载

只需运行 `./setup uninstall` 即可，但请注意该脚本并不完美，且已不再积极维护，因此在响应提示时需要谨慎选择。
