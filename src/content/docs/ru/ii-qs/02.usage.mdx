## 1.1 Обзор

- Открывается с помощью `Super` или `Super`+`Tab`
- Можно перемещать окна между пространствами
- Нажмите средней кнопкой мыши, чтобы закрыть окно

## 1.2 Лаунчер

- Соединен с обзором. Просто начните печатать.
- Результаты могут включать:
  - Поиск приложений
  - Математические вычисления
  - [Префикс: `/`] Действия лаунчера
  - [Префикс: `;`] История буфера обмена
  - [Префикс: `:`] Эмодзи
  - Дополнительные опции для каждого запроса: запустить команду, веб-поиск

- Не-самодокументированные действия лаунчера:
  - /superpaste КОЛ-ВО[i]: Вставляет КОЛ-ВО последних скопированных элементов. Добавьте `i` для изображений.
  - /todo ЗАДАЧА: Добавить в Todo-лист

## 1.3 Верхняя панель

- Управление медиа:
  - СКМ: Играть/Пауза
  - ПКМ: Следующий трек
  - Mouse4/Mouse5: Следующий/предыдущий трек
- Яркость: Прокрутка в левом верхнем углу
- Громкость: Прокрутка в правом верхнем углу
- Пространства:
  - Mouse4, чтобы открыть или закрыть специальное пространство
  - ПКМ, чтобы открыть обзор

# 2 Левая панель

## 2.1 Основное

- Автодополнение по Tab во вкладках ИИ и Аниме
  - Когда появляется подсказка, выберите нужную стрелочками вверх/вниз, а потом подтвердите выбор нажав Tab.
- Хотите забрать ассистента с собой? Чтобы превратить панель в окно, нажмите `Super`+`Alt`+`A`.

## 2.2 ИИ чат

- Напишите `/model`, чтобы выбрать модель. Локальные Ollama-модели подтягиваются автоматически.
- Напишите `/key`, чтобы получить инструкцию, где взять API-ключ для онлайн-моделей.
- На текущий момент, Gemini с Инструментами может редактировать настройки. Вы можете сказать, например: "скрой иконки приложений на моем рабочем пространстве".

## 2.3 Переводчик

Работает через `trans` в терминале. Больше нечего сказать...

## 2.4 Аниме

- Напишите `/mode`, чтобы посмотреть доступные провайдеры.
- Напишите 1 или больше тегов для поиска. При вводе могут появиться подсказки.
- Напишите число, чтобы выбрать, какую страницу результатов смотреть

# 3 Обои и цвета

- Чтобы изменить обои, нажмите `Ctrl`+`Super`+`T` и выберите изображение. Это:
  - Изменит цвета виджетов и QT- и GTK- приложений
  - Изменит позиции часов на экране, переместив их в подходящее место (пока хорошо работает только с одним монитором)

# 4 Скриншоты

- `Super`+`Shift`+`S`: Выбор области экрана | левая кнопка, чтобы скопировать в буфер обмена, правая кнопка, чтобы редактировать
- `Super`+`Shift`+`A`: Поиск через Google Lens
- `Super`+`Shift`+`T`: Распознавание текста

# 5 Перевод экрана

Переводит текст на экране через Cloud Vision и Translation API от Google. В этом разделе описан (в какой-то степени сложный) процесс настройки

## 5.1 Подготовка

- У вас есть Google-аккаунт
- Хотя, если вы не оформляете подписок, эти функции бесплатны, Google требует карту для противодействия мошенничеству. Виртуальная карта с одним центом должна подойти

## 5.2 Настройка

- Перейдите в [Google Cloud Console](https://console.cloud.google.com/) и создайте или выберите проект. Это может быть тот же проект, который вы создали для получения ключа Gemini для левой панели.
- Включите следующие API:
  - [Cloud Vision API](https://console.cloud.google.com/marketplace/product/google/vision.googleapis.com)
  - [Cloud Translation API](https://console.cloud.google.com/marketplace/product/google/translate.googleapis.com)
- В меню перейдите в раздел **IAM & Admin** > **Service Accounts**, на жмите Create service account, затем:
  - Шаг 1: Введите названия. Не важно, какие именно
  - Шаг 2: Права: создайте роль **Service Usage Consumer** и нажмите **Continue**
  - Шаг 3: Нажмите Done
- Вы должны увидеть служебный аккаунт в списке. Откройте меню с тремя точками справа, а затем нажмите **Manage keys**
  - Нажмите **Add key** > **Create new key**
  - Выберите тип ключа JSON (как по умолчанию) и нажмите **Create**. Скачается файл JSON
- Откройте скачанный файл JSON и скопируйте его содержимое
- Откройте переводчик, нажав `Super`+`Shift`+`T`, затем:
  - Нажмите на кнопку с ключом снизу экрана
  - Вставьте скопированный ключ и нажмите Enter. Перевод заработает
- После этого вы можете удалить JSON файл. Ключ будет безопасно храниться в сервисе хранилища ключей
  - Примечание: В данный момент, чтобы посмотреть его снова, необходимо будет искать его в хранилище ключей в разделе "illogical-impulse Safe Storage"

# 6 Группы пространств

## 6.1 Навигация и управление пространствами

Если 10 пространств недостаточно, вы можете использовать столько пространств, сколько разрешает Hyprland. Просто скрольте на соответствующем виджете (или используйте бинды) для переключения на следующую группу.

Бинды и виджеты работают бесшовно на всех пространствах. Вам не надо ничего дополнительно настраивать, если вы не меняете количество пространств в группе.

- **Навигация / перемещение пространств в пределах группы**: Работают стандартные бинды Hyprland.
  - `Super` + `2` → рабочее пространство **12**, если вы в пространстве **11-20 (группа 2)**.
  - `Super` + `Alt` + `3` перемещает окно в фокусе → workspace **23**, если вы в пространстве **21-30 (группа 3)**.

- **Навигация между группами**:
  - Можете добавить следующие бинды в конфиг Hyprland, чтобы переключаться между соседними группами
    ```ini title="~/.config/hypr/custom/keybinds.conf"
    bind = SUPER ALT, Z, workspace, r-10
    bind = SUPER ALT, X, workspace, r+10
    ```
- **Обзор** и **Пространства на панели** адаптируются автоматически. Они актуальны для текущей группы.

:::tip
Если вы хотите менять бинды или добавить [больше функций](https://wiki.hypr.land/Configuring/Dispatchers/) для навигации по пространствам, используйте скрипт `~/.config/hypr/hyprland/scripts/workspace_action.sh`, а не `hyprctl dispatch`. Он правильно определяет текущую группу пространств, используя ID активного..
:::

## 6.2 Работа с несколькими мониторами

Используйте эти методы для эффективной работы с несколькими мониторами:

- Используйте группу 1 (пространства 1-10) на главном мониторе и группу 2 (пространства 11-20) на втором:
  - При включении, на втором мониторе переключитесь на другую группу (напр. пространство 11) используя `Super` + `0` и `Ctrl` + `Super` + `Right`.
  - В таком случае виджет пространств и обзор также будут раздельными.
- Используйте [бинды пространств](https://wiki.hypr.land/Configuring/Workspace-Rules/#rules) чтобы всегда располагать определенные пространства на определенном мониторе. Получить все названия мониторов можно через `hyprctl monitors | grep Monitor`.

  ```ini title="~/.config/custom/general.conf"
  # Закрепить пространства 1-10 (группа 1) на основном мониторе
  workspace = 1, monitor:eDP-1, default:true
  workspace = 2, monitor:eDP-1, default:true
  ...

  # Закрепить пространства 11-20 (группа 2) на втором мониторе
  workspace = 11, monitor:HDMI-A-1, default:true
  workspace = 12, monitor:HDMI-A-1, default:true
  ...
  ```

- Когда два пространства на двух мониторах в фокусе, можно перемещать окна и фокус на них (используя `Super` + `Left/Right/Up/Down` или `Super` + `Shift` + `Left/Right/Up/Down`).

:::note
Группы пространств не поддерживаются нативно ни Hyprland, ни системами виджетов/статус барами. Этот конфиг достигает такой работы благодаря манипуляциям со стандартными диспетчерами.
:::
