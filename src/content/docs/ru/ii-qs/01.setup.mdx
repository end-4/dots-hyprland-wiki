# Установка

:::caution[Прежде чем мы начнем]

- **Только пользовательская настройка**. Вы можете сами установить другой софт для базовой функциональности вашей системы.
- **Нет необходимости полностью переустанавливать систему.**
  :::

:::note[Поддержка дистрибутивов]

- Автоматическая установка: смотрите раздел [Автоматическая установка](#автоматическая-установка) ниже.
- Ручная установка: смотрите раздел [Ручная установка](#ручная-установка) ниже.
- Форки: смотрите раздел [Форки](#форки) ниже.
  :::

:::tip[Переход с AGS версии]
_Пропустите, если не переходите с версии на AGS. Или разберитесь сами, если у вас не Arch Linux_

Версия на Quickshell должна работать прямо после установки, но можете удалить старые пакеты перед установкой новых:

```sh
sudo pacman -Rs $(pacman -Q | grep "illogical-impulse-" | awk '{print $1}')
```

потом устанавливайте.
:::

## Автоматическая установка

:::note[Поддержка дистрибутивов]
Скрипт автоматически определяет дистрибутив. Коротко о поддержке дистрибутивов:

- **Поддержка от разработчиков:**
  - [Arch Linux](https://archlinux.org/) и [EndeavourOS](https://endeavouros.com/) поддерживаются как пользователями, так и разработчиками. Arch более минималистичен, в то время как EndeavourOS предлагает более простую установку.
  - [CachyOS](https://cachyos.org/) использует неофициальные репозитории Arch, но поскольку разрабатывается с поддержкой AUR, все должно работать.
  - Другие [дистрибутивы на базе Arch](https://wiki.archlinux.org/title/Arch-based_distributions) используют свои репозитории и поддерживаются только частично. Особенно [Manjaro](https://manjaro.org/), который известен плохой совместимостью с AUR.
- **Поддержка от комьюнити:**
  - [Gentoo](https://www.gentoo.org/) (см. [sdata/dist-gentoo/README.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/dist-gentoo/README.md) для дополнительной информации).
  - [Fedora](https://fedoraproject.org/) (см. [sdata/dist-fedora/README.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/dist-fedora/README.md) для дополнительной информации).
  - Другие дистрибутивы: Мы рады вашим вкладам! Смотрите [эту страницу](../../dev/inst-script/#help-wanted), если хотите помочь.
- **В работе:**
  - `dist-nix`, находящийся в разработке создан для поддержки всех дистрибутивов. Для дистрибутивов, которые используют старые пакеты, например, [Debian](https://www.debian.org/), это фактически единственный способ установить illogical-impulse. Смотрите [#2382](https://github.com/end-4/dots-hyprland/discussions/2382), чтобы попробовать.
    :::

Выберите одну из опций ниже, затем следуйте инструкции.

:::caution[В чем разница?]

- Опция 1 - обертка над Опцией 2, но вам может быть неудобно запускать онлайн-скрипт прямо на своей рабочей системе.
- Несмотря на ваш выбор, вам следует ознакомиться и понять Опцию 2, потому что это база, которую должен знать любой пользователь Hyprland.
  :::

<Tabs>
<TabItem label="Опция 1: Онлайн-скрипт">

Запустите команды ниже:

```bash
bash <(curl -s https://ii.clsty.link/get)
```

Сначала репозиторий клонируется в `~/.cache/dots-hyprland`, а потом запускается `./setup install`.

:::tip[Дополнительные опции]
Если хотите клонировать в другое место, просто добавьте путь, например:

```bash
bash <(curl -s https://ii.clsty.link/get) ~/Downloads/dots-hyprland
```

Если хотите минимальную установку, предоставляющую в основном, только Hyprland и Quickshell, добавьте опцию --core вот так:

```bash
bash <(curl -s https://ii.clsty.link/get) -- --core
```

Чтобы использовать эти опции вместе, напишите:

```bash
bash <(curl -s https://ii.clsty.link/get) ~/Downloads/dots-hyprland -- --core
```

:::

</TabItem>

Запустите команды ниже:

```bash
cd ~/.cache                                      # Не важно, в какой директории
git clone https://github.com/end-4/dots-hyprland # Клонирование репозиторий
cd dots-hyprland                                 # Перемещение в рабочую директорию
./setup install                                  # Запуск скрипта
```

:::tip[Минимальная установка]
Если хотите минимальную установку, предоставляющую в основном, только Hyprland и Quickshell, запускайте `./setup install --core`
:::

:::tip[Использование команды install]
Подкоманда `install` скрипта `./setup` автономная и интерактивная.

Вы можете сколько угодно раз запускать `./setup install` и выбирать только необходимые действия.

Это полезно, если вы хотите продолжить неудачную установку или выполнить обновление

## Ручная установка

Поддержка: по сути, любой дистрибутив.

:::caution
Этот раздел нужен только для демонстрации.

Крайне не рекомендуется к использованию, только потратите свое время.
:::

### Клонируем и перемещаемся в репозиторий

```bash
t=~/.cache/dots-hyprland   # Не будем оставлять мусор в домашней папке
git clone https://github.com/end-4/dots-hyprland.git "$t" --filter=blob:none --recurse-submodules
cd "$t"
```

### Установка зависимостей

Для Arch-дистрибутивов:

- Установите все пакеты. Их список находится в массиве `depends` в `./sdata/dist-arch/*/PKGBUILD`. Просто используйте инструмент для AUR, такой как `yay`.
- Так же вы можете установить альтернативы, если знаете таковые.

Для не-Arch дистрибутивов:

- Найдите эквиваленты пакетов, список которых в [deps-info.md](https://github.com/end-4/dots-hyprland/blob/main/sdata/deps-info.md) и установите их.
- Если вы успешно установили это все на каком-нибудь не-Arch дистрибутиве, можете отправить PR. Смотрите [эту страницу](../../dev/inst-script/#help-wanted) для подробностей.

### Работа с правами/сервисами и т.д.

Ознакомьтесь с содержимым `sdata/subcmd-install/2.setups.sh`.

- Если вам кажется, что все хорошо, напишите `./setup install-setups`, чтобы запустить скрипт.
  - Не запускайте `sdata/subcmd-install/2.setups.sh` напрямую, потому что он подразумевает запуск только через `./setup`.
- Или запускайте команды из скрипта по отдельности со своими изменениями.

### Копирование файлов конфигурации

- Скопируйте содержимое `dots` в вашу папку `$HOME/` (сначала проверьте, потому что это может перезаписать ваши конфигурации)
- Или прочитайте `sdata/subcmd-install/3.files.sh`.
  - Если все в порядке, запустите `./setup install-files`, что запустит этот скрипт.
  - Не запускайте `sdata/subcmd-install/3.files.sh` напрямую, потому что он подразумевает запуск только через `./setup`.

### Завершение

- Как только закончите, войдите в Hyprland
  - `Ctrl`+`Super`+`T` чтобы выбрать обои.
  - `Super`+`/` чтобы открыть список сочетаний клавиш. Развлекайтесь!

# После установки

## Не совсем опциональные настройки

### Решение возможных конфликтов с демонами уведомлений

- Демоны, такие как `dunst` и `mako`, которые могли быть в вашей системе раньше, могут конфликтовать с реализацией Quickshell. Рекомендуется их удалить, если они не используются где-либо еще.

### Решение конфликта AUR пакетов (только Arch-репозитории)

Проблема: см. [этот комментарий](https://github.com/end-4/dots-hyprland/discussions/3165#discussion-9788226)

Решение: Измените файл `/etc/pacman.conf` (мы не изменяем его автоматически, так как это опасно) и добавьте:

```conf
IgnoreGroup=illogical-impulse
```

Также смотрите [Arch Wiki - Pacman](https://wiki.archlinux.org/title/Pacman#Skip_package_group_from_being_upgraded) чтобы узнать больше о настройке Pacman.

Добавьте эти строки в `~/.zshrc` чтобы включить цветовую схему ZSH:


## Запуск Hyprland

- Чтобы запустить Hyprland, вы можете использовать DM (дисплей-менеджер) или просто `tty`.
- Hyprland wiki рекомендует запускать через сессию, управляемую uswm, но мы не рекомендуем. Это не сломает настройки, но может занести мусор из автозагрузок других окружений (например, дубликатов окон аутентификации)

Смотрите [Hyprland wiki](https://wiki.hypr.land/Getting-Started/Master-Tutorial/#launching-hyprland) для подробностей. Ниже есть дополнительные советы.

### Как автоматически запускать Hyprland сразу после авторизации в `tty1`?

Для ZSH и BASH добавьте следующую строку **в конец** файла `~/.zshrc` или `~/.bashrc`:

```bash
source ~/.config/zshrc.d/auto-Hypr.sh
```

Для FISH, добавьте следующую строку **в конец** файла `~/.config/fish/config.fish`:

```fish
source ~/.config/fish/auto-Hypr.fish
```

P.S. Рекомендуется отключить дисплей-менеджер, если хотите запускать через tty.

### Я новичок. Что такое tty или DM?

Вот краткое описание, которое позволит вам все настроить, но **не конкретная информация**.

Вы могли знать о `tty` как о "базе" Linux-системы.
Обычно есть 7 `tty`: от `tty1` до `tty7`. Вы можете нажать `Ctrl+Alt+F<n>`, чтобы переключиться на `tty<n>`, затем потребуется войти.

После входа есть возможность запустить графическое окружение, такое как `Hyprland`.
Обычно графическое окружение может быть запущено только **после** входа.

Но что делать, если мы хотим использовать графический интерфейс **для самого** входа?

На помощь приходит DM (дисплей-менеджер, который также называют "LM", т.е. Login Manager - менеджер входа).

- Часто используемые DM:
  - `sddm`: Обычно используется с KDE Plasma.
  - `gdm`: Обычно используется с Gnome.
- Они включены на уровне системы, так что запускаются вместе с ней (до входа).
  - На дистрибутивах с systemd, DM обычно включается как systemd-сервис. Запустите следующую команду, чтобы узнать какой дисплей-менеджер активен.
    ```bash
    grep 'ExecStart=' /etc/systemd/system/display-manager.service
    ```
    Если вывод `No such file or directory`, значит никакой DM не активен, или у вас не дистрибутив с systemd.
- DM предоставляет возможность входа и выбора графического окружения (например, Hyprland).
  - Как DM узнает какие окружения доступны?
    - Обычно, он ищет в `/usr/share/xsessions` для X11 и в `/usr/share/wayland-sessions` для Wayland.
    - Desktop-файлы в этих директориях содержат информацию об окружениях.

# Обновление

:::note[TL;DR]
Если вы использовали онлайн-скрипт на Arch-дистрибутиве, обновление можно выполнить командой:

```bash
cd ~/.cache/dots-hyprland && git stash && git pull && ./setup install
```

:::

Пока у нас нет автоматической системы проверки обновлений. Скрипт установки разработан так, чтобы его можно было запускать сколько угодно раз для обновления после синхронизации обновленных файлов из репозитория.

1. С помощью `cd` перейдите в директорию репозитория. Обычно это `~/.cache/dots-hyprland`, если вы не клонировали его где-либо еще.
2. Запустите `git stash`, чтобы сохранить локальные изменения. В идеальном случае, установка не должна трогать файлы, которыми управляет Git, но она может это делать, так что вам необходимо запускать эту команду.
3. Запустите `git pull`, чтобы подтянуть актуальные обновления из репозитория.
4. Снова запустите `./setup install`.
   - Пропустите шаги, которые вы не хотите повторять (особенно `rsync ...`, потому что это **перезапишет ваши файлы**). Обычно вам может потребоваться `rsync` чтобы синхронизировать `dots/.config/quickshell`.

:::tip
Запустите `./setup install -h` чтобы узнать больше.
:::

:::caution[Если вы выполняли ручную установку]
Так как ручная установка нежелательна и не рекомендуется:

- С помощью `cd` перейдите в директорию репозитория, затем напишите `git pull && git submodule update --init --recursive` чтобы подтянуть изменения.
- Еще раз убедитесь, что все зависимости установлены, поскольку при обновлении могут появиться новые.
- Обновите те файлы, которые хотите. Обычно это папка `dots/.config/quickshell`.
  :::

# Удаление

Просто запустите `./setup uninstall`, но учтите что это работает не идеально и не поддерживается активно, так что будьте внимательны, отвечая на вопросы.
